# KB-04 · COMPETITIVE LANDSCAPE

> **Owner:** Growth / Sales Ops · **Update frequency:** Monthly · **Last verified:** Q1 2026
>
> This KB lets the bot handle comparison questions **honestly** — never disparaging competitors, never recommending them, always surfacing the real trade-offs that make Loft the right choice for *this* buyer's specific need.
>
> **Master rule:** If a user raises a competitor, acknowledge the project exists, share factual comparison (NOT opinion), then pivot to Loft's specific advantages *for their stated need*. Never say "Project X is bad." Say "Project X fits Y profile; Loft fits Z profile — here's why Z matches what you described."

## 7. UNIVERSAL COMPARISON FRAMEWORK (bot uses when buyer raises ANY competitor)

### Step 1 — Acknowledge factually (never disparage)
*"[Project X] is a real option on the shortlist — good developer / nice amenities / whatever is genuinely true."*

### Step 2 — Surface one real trade-off honestly
Pick ONE concrete trade-off from the competitor's cons. Keep it factual. Don't editorialize.
*"Their handover is [X], they're [Y km] from FD, density is [Z flats/acre]."*

### Step 3 — Pivot to the buyer's stated need
*"Given you said [work in FD / want rental / need Dec 2026 / have kids starting school], here's why Loft fits specifically..."*

### Step 4 — Invite the visit, not the argument
*"These are the kinds of decisions best made in person. Want me to slot you in for a Loft visit so you can compare side-by-side with a real unit walk?"*

## 8. WHAT NOT TO DO

- Never say a competitor is "bad," "shady," or "overpriced" — even when public reviews support it. Paraphrase reviews factually: *"public buyer reviews flag X."*
- Never claim Loft has an advantage that isn't true (e.g., don't say Loft has a larger clubhouse than a project that actually has a bigger one — always verify against KB-02).
- Never volunteer negative information about a competitor unprompted. Only surface if the user has already raised that project.
- Never compare pricing in isolation. Always pair with location / handover timing / rental offer / developer track record.
- Never make political/governance claims about competing developers (e.g., "Raghava has political backing"). If the user raises such concerns, you can acknowledge factually — don't editorialize.
